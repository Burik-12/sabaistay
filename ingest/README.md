# Ingest — сборщики источников

## Что есть сейчас (бесплатно)

### `telegram_preview.py` — Telegram через web-превью
- Парсит `t.me/s/<channel>` **без аккаунта, без ключей, без затрат**. Только stdlib.
- Годится для старта и теста парсера на живых данных. Главный канал: `rent_phangan` (~8к, рус).
- Ограничения: только **публичные каналы** (не чаты/группы), ограниченная глубина истории, нет realtime (только повторный запуск).
```bash
python3 telegram_preview.py rent_phangan --pages 5 -o ../data/sample_tg.json
```

## Что добавим позже (платно/с аккаунтом)

### Telegram userbot 24/7 (Telethon) — Фаза 1, ~$5/мес
- Постоянный мониторинг множества каналов **и чатов**, realtime (`events.NewMessage`) + добор по `min_id`.
- Нужны: отдельная SIM/номер (НЕ личный), `api_id`/`api_hash` с my.telegram.org, дешёвый VPS 24/7.
- Гигиена: прогретый аккаунт, задержки, FloodWait-backoff, НЕ трогать списки участников.

### Facebook — Фаза 3, ~$60-110/мес (главный источник)
- **Managed-актор на Apify** (не самопис — Meta ломает разметку раз в 3-6 недель, пусть чинит автор актора):
  - публичные группы + Marketplace «Koh Phangan» по расписанию каждые 3-6 ч;
  - закрытые группы — cookie «расходного» FB-аккаунта (не личного) + резидентный тайский прокси.
- Антибан: отдельный прогретый аккаунт, низкий темп, рандомизация, запасной аккаунт на checkpoint.
- Мониторинг: если прогон вернул 0/мусор → пуш в Telegram «FB сломался».
- Юридика: только факты + ссылка, без PII/ре-хостинга фото (см. `../docs/legal-pdpa.md`).

### Airbnb/Booking — Фаза 5, бесплатно (потом)
- Переиспользуем коллекторы `rental-market-ci` из RW: медианы ADR по районам → бенчмарк «на X% ниже/выше рынка».

## Единый формат на выходе
Все сборщики отдают записи в формате таблицы `raw_post` (`../db/schema.sql`):
`source`, `source_url`, `external_id`, `posted_at`, `raw_text`. Дальше — Claude-парсер
(`../parser/`) → нормализация района (`../data/districts.json`) → дедуп → база.
