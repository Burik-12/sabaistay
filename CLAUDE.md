# CLAUDE.md — SabaiStay

Это **отдельный продукт** Владимира: нейтральный агрегатор-поисковик жилья на Ко Пхангане.
Не путать с риелторской компанией **Right Way** (RW) — у RW отдельный проект по пути
`~/Documents/Claude/Projects/Right Way - сбор компании/`. SabaiStay **не брендирован**
как RW и технически от него **отделён** (свой репо, база, домен). Код-ядро можно
переиспользовать из RW копированием, но приватные данные RW сюда не тащить.

## Как общаться
- На русском, на «ты», дружелюбно и по делу (как в личном проекте Владимира).
- Если данных не хватает — спроси, не выдумывай.

## Что строим (суть)
Источники → парсер → база → витрина с фильтром + бот «подписка на фильтр».
- **Источник №1 — Facebook** (главный, там основная масса объектов): Marketplace + группы аренды. Сбор через managed-актор (Apify) + «расходный» аккаунт + резидентный прокси. Платно.
- **Источник №2 — Telegram** (постоянный мониторинг): Telethon userbot 24/7. Старт — бесплатный сбор через `t.me/s/<channel>` без аккаунта.
- **Позже** — сравнение цен с Airbnb/Booking (медианы по районам, переиспользуем коллекторы RW `rental-market-ci`).

## Главные принципы
1. **Ценность = фильтр.** Всё сводится к чистым фильтруемым полям (район/цена/спальни/тип/удобства/даты). Это ядро, см. `parser/` и `db/schema.sql`.
2. **PDPA-safe (Владимир — оператор в Таиланде).** Храним только **факты объекта + ссылку на оригинал**. Телефоны/имена/фото **не** храним как структурное поле — за контактом пользователь идёт по ссылке. Подробности — `docs/legal-pdpa.md`.
3. **Каждое объявление — ссылка на первоисточник** (`source_url`).
4. **Telegram — надёжный костяк, Facebook — хрупкий но главный.** FB ломается раз в 3-6 недель → managed-актор, а не самопис; мониторинг поломок.

## Сейчас бесплатная фаза
Делаем всё, что без денег: документация, модель данных, схема парсера, газеттир,
бесплатный Telegram-превью сборщик, форк витрины. Платное (прокси/Apify/VPS) —
когда будет бюджет. Текущий план и развилки — `docs/roadmap.md`.
